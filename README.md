# Multimodal Skin Cancer Classification 🔬

This repository contains the study and codebase for a **Multimodal Machine Learning approach** to skin lesion classification, aimed at assisting dermatological diagnostics.

This project is part of a **Scientific Initiation (PIBIC)** research and directly compares the performance of deep learning feature extraction (CNNs) against classical machine learning algorithms on tabular and image data.

## 📊 Dataset: HAM10000

We utilized the widely recognized **HAM10000** ("Human Against Machine with 10000 training images") dataset. It encompasses a large collection of multi-source dermatoscopic images of common pigmented skin lesions with corresponding clinical metadata (Age, Sex, and Localization).

## 🧠 Methodology

The methodology is built upon a dual-branch neural architecture:

1. **Imaging Branch**: Utilizes *EfficientNetB3* pre-trained on ImageNet to extract visual features from the dermatoscopic inputs.
2. **Clinical Branch**: Processes metadata (Age, Sex, Localization) natively through dense layers.
3. **Fusion & Evaluation**: The embeddings from the CNN head are concatenated with the clinical metadata before the final classification. We compare this CNN end-to-end performance against classic classifiers (Random Forest, SVM, XGBoost, Naive Bayes) strictly trained over the robust embeddings generated by the Neural Network.

**Loss Function**: `Categorical Focal Loss` was successfully applied to penalize easily classified examples and target the severe class imbalance inherent to dermatological sets (where benign Nevi dominate over Melanoma/Carcinomas).

## 🚀 Key Fixes

During code review, critical improvements were applied to the testing script to adhere to scientific standards:

- **Zero Data Leakage**: Enforced independent stratified data splits before evaluating feature extractors on classical algorithms.
- **Metric Expansion**: Added AUC (Area Under the Receiver Operating Characteristic Curve) leveraging the One-vs-Rest strategy for the multi-class clinical requirement, completing the accuracy and Macro F1-Score reports.

## 🤝 How to Use

For testing: simply run `notebooks/Teste Multimodal - HAM10000 - Corrigido.ipynb` preferably via **Google Colab** (enabling T4 or L4 GPU). It mounts directly to Google Drive or local storage and automatically evaluates your `modelo_multimodal_final.keras`.
